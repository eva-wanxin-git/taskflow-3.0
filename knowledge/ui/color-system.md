## 使用规范

### 边框标识

**左侧色条**（4px宽）:
- 用于标识卡片类型
- 悬停时加粗到8px
- 示例：事件卡片、记忆卡片

```css
.card {
    border-left: 4px solid #9e9e9e;
}

.card:hover {
    border-left-width: 8px;
}

.card.category-architecture {
    border-left-color: #5d6d7e;
}
```

### 顶部色条

**标题下方**（2-3px粗线）:
- 用于区分主要区块
- 黑色表示标准模块
- 特殊色表示特定模块

```css
.section {
    border-top: 2px solid #000;
}

.ops-section {
    border-top: 2px solid #7BA882;  /* SRE模块特殊色 */
}
```

### 背景色

**模块背景**:
- 标准模块：白色 `#FFFFFF`
- SRE模块：淡绿半透明 `rgba(123, 168, 130, 0.03)`
- 输入框：浅灰 `#fafafa`
- 代码块：浅灰 `#f5f5f5`

## 对比度要求

### 可读性标准

- 主要文字对比度 ≥ 7:1 (AAA级)
- 次要文字对比度 ≥ 4.5:1 (AA级)
- 图标对比度 ≥ 3:1

### 检查工具

使用Chrome DevTools的Contrast Ratio检查器验证对比度

---

## 色彩使用禁忌

### ❌ 避免使用

- 鲜艳的纯色（不符合工业美学）
- 渐变色（保持简洁）
- 阴影过重（扁平化设计）
- 过多色彩（保持克制）

### ✅ 推荐使用

- 黑白为主
- 灰度过渡
- 功能色点缀
- 高对比度
